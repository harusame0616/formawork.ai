# タスク完了チェックリスト

## タスク完了条件（必須）

タスクを完了とする前に、以下をすべて確認すること：

### 1. バリデーションチェック
```bash
pnpm validate:check
```

以下がすべてパスすること：
- [ ] lint エラーがないこと
- [ ] format が適用されていること
- [ ] デッドコード（未使用のエクスポートなど）がないこと
- [ ] スペルミスがないこと
- [ ] 型エラーがないこと

### 2. ランタイム確認
- [ ] ランタイムエラーがないこと
- [ ] 開発サーバーで動作確認（必要に応じて）

### 3. 自動修正（エラーがある場合）
```bash
pnpm validate:fix
```

## マージ条件（PR マージ前）

タスク完了条件に加えて：

- [ ] ユニットテストがパスしていること
- [ ] コンポーネントテストがパスしていること
- [ ] E2E テストがパスしていること
- [ ] デプロイが成功していること

## テスト実行コマンド

```bash
# ブラウザテスト
pnpm test:browser

# サーバーテスト
pnpm test:server

# E2E テスト
pnpm test:e2e
```

## 注意事項

- 新機能には必ずテストを作成すること
- テストカバレッジ目標: 80% 以上
- 外部ライブラリ使用時は Context7 で正しい情報を取得
- GitHub 操作は CLI ではなく MCP ツールを使用
